How It Works. Simplifed!

This project uses the stripe backend API call to handle payment processing. It does this by integrating the Stripe API in the Python backend and transferring sensitive payment information such as a 'client secret token' which is used to authorize and authenticate the payment elements and confirm transactions through Stripe.

Steps in developing this project

Step 1: Create Django View

Backend view function that renders the page before the checkout page

Step 2: Client-side Fetch Request

Use client-side code to send a fetch request to the backend, requesting a payment intent and a client secret.

Step 3: Create Django View for Payment Intent

Another backend view function that generates a payment intent and sends the data back to the client-side for checkout.

Step 4: Use Client Secret for Stripe Elements

Utilize the client secret obtained from Stripe API to securely instantiate elements for handling sensitive user data.

Step 5: Verify Payment Details with Stripe API

After users enter their payment details, the Stripe API verifies the provided information.
